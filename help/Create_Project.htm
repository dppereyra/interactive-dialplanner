Starting a Project

  1. Start the Interactive Dialplanner development environment and create a new project.
    a. Click File > New Project.  A dialog box appears.

  2. In the Project dialog box, Specify the Project Name, Description, Version, and the extension for the Asterisk Call Flow context.

    The Interactive Dialplanner  allow you to remote update the call flow via SSL.  Your Asterisk PBX should enable the SSH and SFTP feature for the remote update.  To enable this feature, Please check "Using SSL for Remote Update" and specify the host , or IP address of the Asterisk PBX, SSH port number, login user name, password.  The File Path and Command should follow the default setting for the Asterisk.  Please specify these if your Asterisk PBX has a differenct configuration.

    Click OK to finish creating your project.


  3. After creating project, you should create a first call flow page for your call flow project.  The call flow page creates a design area onto which you add the activity chosen in the design phase.  Also you can add more call flow pages if necessary.

    To add the call flow page:
    a, Click Project > Add Call Flow.  A dialog appears.
    b, Specify the name of call flow.  Notes the name of call flow will be the name of context in the Asterisk PBX.


    c, The Call Flow Page appears.  You can drag the Activity in the Toolbox into the Call Flow Page.


  4. Setting Activity Name and properties
    Every activity in the call flow has a unique name that identify the activity and properties.  The name of activity is stored in the "Label" field of the activity.

    The Interactive Dialplanner has two way to change the activity name.
    a, Select the activity on the call flow.  The activity name will show on the properties area.  You can change the value of label field.




    b, Double click the activity.  The dialog box appears.



    Click "OK" to confirm the changes.
  5. Connecting activities
    Activity have a input node on the left-hand side. All activities, with the exception of Hangup SubProcEnd and Goto, have one or more output nodes on the right-hand side.  A call enters the activity through an input node and after the activity finishes processing, it directs the call flow through the output node to the next activity.

    To connect one activity to another:
    a, Select an output node.
    b, Drag the mouse to the input node of the activity that performs the next action.
    c, Release the mouse button. The connection becomes permanent.
