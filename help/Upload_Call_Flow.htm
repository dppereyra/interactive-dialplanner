Running your Compiled Call Flow in Asterisk

Using Elastix 2.0 as a example

Assume the callflow will be compiled and the dialplan had been uploaded.  The dialplan file name is "Authentication Example.cfd.conf", and the name of context is "Main_Flow".

  1. Edit extensions_custom.conf.
    Add #include into the extensions_custom.comf



  2. Add a custom extension device.
    For example, we assign 7900 as a device number for the extension.  The "dial" field will use the Local channel feature to point our dial plan context.



  3. In the Inbound Route, assign DID number for the extension 7900.



  4. Click "Apply Configuration Changes Here" to reload the system.


  5. Click PBX > Tools > Asterisk-Cli.  Run the command "dialplan reload".


  6. You can dial into Asterisk by the DID to test the compiled dial plan.
